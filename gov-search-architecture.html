// REQUEST FLOW — ทีละ step

1
User Query
ส่งคำถามเข้า Orchestrator
2
Semantic Routing
embed query → vector search บน agency catalog → ได้ top-3~5 หน่วยงาน
3
Dynamic Tool Inject
สร้าง tool schema เฉพาะ 3~5 หน่วยงานนั้น ใส่เข้า LLM prompt
4
LLM Tool Call
Claude เรียก tool ที่เหมาะ อาจเรียก parallel ถ้าต้องหลายหน่วยงาน
5
Adapter Fetch
REST call หรือ PDF vector search ตาม adapter ของหน่วยงาน
6
Synthesize
Claude รวมผลจากทุกแหล่ง → ตอบ user พร้อม source citation
🔑 Key Design Decision
ไม่ใส่ tool ครบ 30+ ใน prompt
ใช้ Semantic Router คัด top-K ก่อน
→ LLM เห็นแค่ tool ที่เกี่ยวข้องเท่านั้น
📐 Scalability
เพิ่มหน่วยงานใหม่ = แค่ register ใน Agency Registry
ไม่ต้องแก้ Orchestrator หรือ prompt
⚡ PDF Pipeline
PDF ถูก index ล่วงหน้าเป็น vector store
ตอน query ทำ semantic search บน chunk
ไม่ต้องแปลง PDF ตอน runtime
⚠ จุดที่ต้องระวัง: Agency catalog ต้องมี metadata ที่ดี (domain tags, keywords) เพราะ Semantic Router ใช้ตัวนี้ตัดสินใจ ถ้า metadata ไม่ดี routing จะผิดพลาดได้
