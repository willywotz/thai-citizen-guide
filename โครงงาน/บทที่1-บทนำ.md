# บทที่ 1 บทนำ

## 1.1 ความเป็นมาและความสำคัญของปัญหา

ในปัจจุบัน หน่วยงานภาครัฐของประเทศไทยมีจำนวนมาก แต่ละหน่วยงานมีระบบสารสนเทศและช่องทางการให้บริการข้อมูลแก่ประชาชนที่แตกต่างกัน ไม่ว่าจะเป็นสำนักงานคณะกรรมการอาหารและยา (อย.) กรมสรรพากร กรมการปกครอง หรือกรมที่ดิน ทำให้ประชาชนที่ต้องการสอบถามข้อมูลจากหลายหน่วยงานพร้อมกัน จำเป็นต้องเข้าไปค้นหาในเว็บไซต์หรือติดต่อแต่ละหน่วยงานแยกกัน ซึ่งเป็นกระบวนการที่ใช้เวลาและสร้างความไม่สะดวก

นอกจากนี้ เทคโนโลยี AI โดยเฉพาะ Large Language Model (LLM) ได้รับการพัฒนาอย่างก้าวกระโดด สามารถเข้าใจและตอบคำถามภาษาธรรมชาติได้อย่างมีประสิทธิภาพ อย่างไรก็ตาม LLM ทั่วไปมีข้อจำกัดในเรื่องข้อมูลที่ไม่ทันสมัยและไม่สามารถเข้าถึงข้อมูลเฉพาะของหน่วยงานภาครัฐได้โดยตรง จึงเกิดแนวคิดในการพัฒนาระบบ AI Chatbot ที่ทำหน้าที่เป็นศูนย์กลาง (Portal) ในการรับคำถามจากประชาชน แล้วส่งต่อไปยังหน่วยงานที่เกี่ยวข้อง จากนั้นนำคำตอบที่ได้มาสังเคราะห์เป็นคำตอบเดียวที่ครอบคลุมและเข้าใจง่าย

แนวคิด Multi-Agent System ถูกนำมาใช้ในการออกแบบระบบนี้ โดยระบบจะมี Agent สำหรับแต่ละหน่วยงาน ทำหน้าที่ดึงข้อมูลจากแหล่งข้อมูลของหน่วยงานนั้น ๆ แล้วส่งกลับมาให้ Agent หลักทำหน้าที่สังเคราะห์คำตอบ วิธีการนี้ทำให้ระบบมีความยืดหยุ่นสูง สามารถเพิ่มหรือลดหน่วยงานได้โดยไม่กระทบต่อระบบโดยรวม

ด้วยเหตุผลดังกล่าว คณะผู้จัดทำจึงพัฒนา "AI Chatbot Portal สำหรับสังเคราะห์คำตอบจากหน่วยงานภาครัฐ" เพื่อเป็นต้นแบบของระบบที่ช่วยให้ประชาชนสามารถเข้าถึงข้อมูลภาครัฐได้อย่างสะดวกรวดเร็วผ่าน Chatbot เพียงจุดเดียว

## 1.2 วัตถุประสงค์ของโครงงาน

1. เพื่อพัฒนา Web Application แบบ Full-Stack สำหรับ AI Chatbot ที่สามารถรับคำถามจากประชาชนแล้วสังเคราะห์คำตอบจากหลายหน่วยงานภาครัฐ
2. เพื่อพัฒนาระบบ Multi-Agent Routing ที่สามารถตรวจจับหน่วยงานที่เกี่ยวข้องจากคำถามภาษาไทย และส่งคำถามไปยังหน่วยงานเหล่านั้นแบบ Parallel
3. เพื่อพัฒนาระบบจัดการหน่วยงาน (Agency Management) ที่รองรับการเชื่อมต่อหลายรูปแบบ ได้แก่ REST API, MCP (Model Context Protocol) และ A2A (Agent-to-Agent)
4. เพื่อพัฒนา Dashboard วิเคราะห์สถิติการใช้งาน ได้แก่ จำนวนคำถาม สถิติการใช้งานแต่ละหน่วยงาน และ Feedback ของผู้ใช้

## 1.3 ขอบเขตของโครงงาน

ระบบที่พัฒนาขึ้นมีขอบเขตดังนี้:

**ด้าน Frontend:**
- Public Portal สำหรับประชาชนทั่วไป ประกอบด้วย Landing Page แสดงหน่วยงานที่ให้บริการ และ Chat Interface สำหรับสอบถามข้อมูล
- Admin Portal สำหรับเจ้าหน้าที่ ประกอบด้วย Dashboard, Agency Management (CRUD), Chat Interface ภายใน, ประวัติการสนทนา
- ระบบ Authentication ด้วย JWT สำหรับเจ้าหน้าที่ (Login, Signup, Reset Password)
- รองรับ Responsive Design

**ด้าน Backend:**
- REST API ด้วย FastAPI (Python) รองรับ Async
- ฐานข้อมูล PostgreSQL ผ่าน Tortoise ORM
- ระบบ Chat ที่ใช้ Keyword-based Agency Detection สำหรับภาษาไทย
- Multi-Agent Routing ด้วย LangGraph สำหรับการ route คำถามไปยังหน่วยงานที่เกี่ยวข้อง
- สังเคราะห์คำตอบผ่าน LLM (OpenThai GPT ผ่าน AI Gateway)
- MCP Server ด้วย FastMCP สำหรับเปิดเผย Agency Metadata ให้ LLM Client ภายนอก

**ด้าน Infrastructure:**
- Docker Compose สำหรับ Container Orchestration (PostgreSQL, Backend, Nginx)
- Nginx เป็น Reverse Proxy

**หน่วยงานตัวอย่างที่รองรับ:**
- สำนักงานคณะกรรมการอาหารและยา (อย.)
- กรมสรรพากร
- กรมการปกครอง
- กรมที่ดิน

**สิ่งที่ไม่อยู่ในขอบเขต:**
- การเชื่อมต่อกับ API จริงของหน่วยงานภาครัฐ (ใช้ข้อมูลจำลอง)
- ระบบ Voice Input/Output
- ระบบ Caching ระดับ Production
- การรองรับหลายภาษา (รองรับเฉพาะภาษาไทยเป็นหลัก)

## 1.4 ประโยชน์ที่คาดว่าจะได้รับ

1. ประชาชนสามารถสอบถามข้อมูลจากหลายหน่วยงานภาครัฐได้ผ่านจุดเดียว โดยไม่ต้องเข้าเว็บไซต์แต่ละหน่วยงานแยกกัน
2. ลดภาระงานของเจ้าหน้าที่ในการตอบคำถามที่ซ้ำ ๆ จากประชาชน
3. หน่วยงานภาครัฐสามารถวิเคราะห์ความต้องการของประชาชนผ่าน Dashboard และ Feedback Analytics
4. เป็นต้นแบบ (Prototype) ของระบบ AI Chatbot ภาครัฐที่สามารถต่อยอดเพื่อใช้งานจริงได้ในอนาคต
5. ได้เรียนรู้และประยุกต์ใช้เทคโนโลยีสมัยใหม่ ได้แก่ LangGraph, MCP Protocol, FastAPI, React with TypeScript

## 1.5 นิยามศัพท์เฉพาะ

| คำศัพท์ | ความหมาย |
|---------|----------|
| AI Chatbot | โปรแกรมที่ใช้ปัญญาประดิษฐ์ในการสนทนาและตอบคำถามกับมนุษย์ผ่านข้อความ |
| LLM (Large Language Model) | โมเดลภาษาขนาดใหญ่ที่ถูกฝึกด้วยข้อมูลจำนวนมหาศาล สามารถเข้าใจและสร้างข้อความภาษาธรรมชาติได้ |
| Multi-Agent System | ระบบที่ประกอบด้วย Agent หลายตัวทำงานร่วมกัน แต่ละตัวมีหน้าที่เฉพาะทาง |
| MCP (Model Context Protocol) | โปรโตคอลมาตรฐานสำหรับการเชื่อมต่อ LLM กับแหล่งข้อมูลภายนอก พัฒนาโดย Anthropic |
| A2A (Agent-to-Agent) | โปรโตคอลสำหรับการสื่อสารระหว่าง AI Agent ต่าง ๆ พัฒนาโดย Google |
| LangGraph | Framework สำหรับสร้าง Multi-Agent Workflow ที่ซับซ้อน สร้างบน LangChain |
| FastAPI | Web Framework ของ Python ที่รองรับ Async และสร้าง API ได้รวดเร็ว |
| REST API | รูปแบบการออกแบบ API ตามหลัก Representational State Transfer |
| JWT (JSON Web Token) | มาตรฐานสำหรับการยืนยันตัวตนแบบ Token-based |
| ORM (Object-Relational Mapping) | เทคนิคการแปลงข้อมูลระหว่าง Object ในโปรแกรมกับตารางในฐานข้อมูล |
| Docker | แพลตฟอร์มสำหรับสร้างและรัน Container เพื่อให้แอปพลิเคชันทำงานได้เหมือนกันในทุกสภาพแวดล้อม |
| Nginx | Web Server ที่ทำหน้าที่เป็น Reverse Proxy ส่งต่อ Request ไปยัง Backend |
| SSE (Server-Sent Events) | เทคโนโลยีสำหรับส่งข้อมูลจาก Server ไปยัง Client แบบ Real-time ทางเดียว |
