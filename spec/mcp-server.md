# ข้อกำหนดการพัฒนาระบบ MCP Server (Model Context Protocol) 

เอกสารฉบับนี้จัดทำขึ้นสำหรับผู้พัฒนา MCP Server เพื่อกำหนดมาตรฐานการเชื่อมต่อกับระบบ AI Orchestrator (OneChat) โดยมีข้อกำหนดทั้งในส่วนของการเชื่อมต่อและโครงสร้างข้อมูลดังนี้:

---

## 1. การเชื่อมต่อตามมาตรฐาน MCP (SSE Transport)

เพื่อให้ระบบสามารถทำงานได้อย่างสมบูรณ์ภายใต้มาตรฐาน Model Context Protocol การรับส่งข้อมูลผ่าน HTTP จะต้องใช้การทำงานแบบ Server-Sent Events (SSE) ตามกฎเกณฑ์ดังต่อไปนี้:

- **การเปิดการเชื่อมต่อ (GET Request):** 
  - จุดรับการเชื่อมต่อ (เช่น `GET /sse`) จะต้องรับ Request ที่มี Header Accept: text/event-stream ได้ทันที
  - **ห้าม**บังคับใช้พารามิเตอร์ session_id หรือบังคับให้ส่ง Token ก่อนเปิดสตรีม 
  - Server ต้องทำหน้าที่สร้าง Session ID ให้เอง และพ่นรหัสผ่าน Event กลับมาทาง SSE Stream
- **การส่งคำสั่ง (POST Request):** 
  - การรับคำสั่งทำงานใดๆ (เช่น การเรียกดูรายชื่อผ่าน list_resources หรือ `list_tools`) ต้องแยกมาใช้ HTTP POST คู่กับ URL ที่รับข้อความ เช่น POST /messages?sessionId=... อ้างอิงตาม Session ID ที่ Server ออกให้ไปแล้ว

---

## 2. โครงสร้างข้อมูลรายชื่อหน่วยงาน (Agency Catalog)

ระบบนี้ออกแบบโดยประยุกต์ใช้ความสามารถหมวด **Resources (ทรัพยากร/ดึงข้อมูลอ้างอิง)** ของสถาปัตยกรรม MCP ซึ่งเป็นหมวดที่ออกแบบมาเพื่อให้ AI สามารถเปิดอ่านแฟ้มข้อมูลตายตัว หรือฐานข้อมูลภายนอกได้โดยเฉพาะ

ดังนั้น เมื่อระบบ Orchestrator สั่งขออ่านข้อมูลจาก Resource ที่มีชื่อ URI ว่า agencies://list ตัวเซิร์ฟเวอร์จะต้องรับคำสั่งและคืนค่ากลับมาในรูปแบบ JSON Payload ตามโครงสร้างตัวอย่างด้านล่างนี้:

json
{
  "agencies": [
    {
      "name": "กรมการปกครอง",
      "description": "ระบบตรวจสอบข้อมูลทะเบียนราษฎร์ บัตรประชาชน และงานปกครอง",
      "endpoint_url": "http://203.154.130.166/dopa/chat",
      
      "expected_payload": {
        "session_id": "",
        "query": "{ข้อความคำถามจากผู้ใช้}"
      }
    },
    {
      "name": "กรมที่ดิน",
      "description": "ระบบสอบถามข้อมูลที่ดิน โฉนด การจดทะเบียนสิทธิและนิติกรรม",
      "endpoint_url": "http://203.154.130.166/dol/chat",
      
      "expected_payload": {
        "session_id": "",
        "query": "{ข้อความคำถามจากผู้ใช้}"
      }
    }
  ],
  "total": 2
}

### คำอธิบายพารามิเตอร์ (Fields)
- **`name`** *(string)*: ชื่อของหน่วยงานปลายทาง (สงวนสิทธิ์ชื่อห้ามซ้ำกัน)
- **`description`** *(string)*: คำอธิบายอย่างละเอียดเกี่ยวกับขอบเขตข้อมูลที่หน่วยงานรับผิดชอบ (AI Agent จะกวาดอ่านข้อมูลส่วนนี้เพื่อตัดสินใจคัดกรองคำถามของ User)
- **`endpoint_url`** *(string)*: URL ปลายทางของ REST API สำหรับหน่วยงานนั้นๆ เพื่อให้ระบบ Orchestrator วิ่งเข้าไปส่งชุดคำถาม (HTTP POST) 
- **`expected_payload`** *(object)*: โครงสร้างแสดงตัวอย่างของข้อมูล (JSON Body) ที่ระบบของแต่ละหน่วยงานคาดหวังให้ประสานงานกลับไป
