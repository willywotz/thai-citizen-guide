// ADAPTER COMPARISON
TYPE รับข้อมูล วิธี query latency ข้อควรระวัง
REST structured JSON HTTP params → parse response เร็ว schema ต่างกันแต่ละหน่วยงาน ต้อง normalize
PDF PDF binary pre-index → vector search ตอน query index ช้า, query เร็ว PDF scan ต้อง OCR, layout ซับซ้อนอาจ extract ผิด
TEXT plain text / HTML scrape → chunk → vector search เร็ว เว็บเปลี่ยน structure บ่อย ต้องมี recrawl schedule
A2A task/message ส่ง task → poll หรือ webhook รับ result ช้า (async) ต้องจัดการ task lifecycle, timeout, error state
MCP tool / resource call MCP tool โดยตรง เร็ว ต้องเชื่อมต่อ MCP server ให้ได้ก่อน (stdio หรือ SSE)
LLM natural language ส่ง query → รับ answer text ขึ้นอยู่กับ model ต้องมี trust score, อาจ hallucinate, ยาก verify
🔑 Key insight: ทุก adapter ต้องคืนข้อมูลในรูปแบบเดียวกัน ({ source, content, score, metadata }) ให้ Orchestrator ก่อนที่จะสังเคราะห์คำตอบ — ความต่างของแต่ละ type ซ่อนอยู่ใน adapter ไม่รั่วออกมาถึง LLM

⚠ LLM adapter ต้องระวังพิเศษ เพราะคำตอบที่ได้มาเป็น generated text แล้ว ไม่ใช่ raw data → ยาก verify → ควรมี trust score ต่ำกว่า adapter อื่นเวลา rank ผลลัพธ์
