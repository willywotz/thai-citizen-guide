# OneChat v5 Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move `POST /api/v1/chat/stream` from OneChat v4 to v5 — relaying the new `summarize` step, persisting and rendering the `summary` + `references[]`, and using `thread_name` as the conversation title — behind a `CHAT_STREAM_VERSION` switch that defaults to `v5`.

**Architecture:** The portal is a proxy: `chat_stream` opens an SSE connection to the upstream OneChat URL and re-emits every event. v5 adds one step event (relayed with zero code change) and three payload fields. Backend work is (a) choosing the upstream URL from config, (b) two new `messages` columns, (c) threading the new fields through `_save_stream_conversation` → `save_turn`. Frontend work is (a) new event/state types, (b) a shared `SummaryCard`, (c) a pure `stripSummaryPrefix` helper so the summary is not rendered twice (upstream's `answer` string already embeds it).

**Tech Stack:** Python 3.12 · FastAPI · Tortoise ORM · aerich · pytest · React 18 · TypeScript · vitest · Tailwind/shadcn.

**Design doc:** `docs/superpowers/specs/2026-07-21-onechat-v5-design.md`
**Upstream contract:** `spec/v5.md`

## Global Constraints

- **TDD is mandatory.** Failing test → confirm it fails → minimal code → confirm it passes → commit. No exceptions.
- **Branch:** all work lands on `feat/onechat-v5` (already created off `dev`). PR targets `dev`. Never push `main`.
- **Prefix every shell command with `rtk`** (token-optimizing proxy), including inside `&&` chains.
- **Never hand-author a migration or carry `MODELS_STATE` forward.** Always regenerate with `aerich migrate` against a DB that has all prior migrations applied. Read `docs/aerich-migrations.md` before Task 2.
- **The new column is `summary_references`, never `references`** — `REFERENCES` is a reserved SQL keyword in Postgres.
- **Do not touch** `POST /chat` / `chat_external` (OneChat v3 sync). v5 defines no sync endpoint.
- Backend tests run against SQLite in-memory via the `db` fixture (`backend/tests/conftest.py`); keep new tests SQLite-portable (no Postgres-only SQL).
- Latest applied migration on `dev` is `21_20260711014939_popular_questions.py`; the new one will be `22_*`.
- After the last task: update `context.md` and commit (project rule in `CLAUDE.md`).

```python
def _stream_upstream() -> tuple[str, str]:
    """Resolve (version, url) for the streaming upstream from CHAT_STREAM_VERSION.

    Unknown values fall back to v5 rather than calling a URL that does not exist,
    so a typo in the /settings override degrades to the default instead of
    breaking chat outright.
    """
    version = (settings.CHAT_STREAM_VERSION or "").strip().lower()
    if version == "v4":
        return "v4", settings.ONECHAT_V4_URL
    if version != "v5":
        logger.warning("Unknown CHAT_STREAM_VERSION %r — falling back to v5", settings.CHAT_STREAM_VERSION)
    return "v5", settings.ONECHAT_V5_URL
```

```ts
/** Matches the `---` horizontal rule OneChat v5 puts between summary and sections. */
const DIVIDER = /\n-{3,}\n/;

/**
 * Returns the section body of a v5 `answer`, with the leading executive summary
 * (and its reference list) removed.
 *
 * v5 composes `answer` as summary → reference list → `---` → sections. The UI
 * renders the summary in its own card, so rendering the whole string as well
 * would show it twice. Splitting on the FIRST divider after the summary keeps
 * any `---` inside raw agency content intact.
 *
 * Never drops agency content: with no summary, a non-matching summary, or no
 * divider, it returns everything that is not the summary itself.
 */
export function stripSummaryPrefix(content: string, summary?: string | null): string {
  if (!summary) return content;
  if (!content.startsWith(summary)) return content;

  const rest = content.slice(summary.length);
  const match = rest.match(DIVIDER);
  if (!match || match.index === undefined) return rest.trim();

  return rest.slice(match.index + match[0].length).trim();
}
```

```tsx
import { Sparkles } from 'lucide-react';
import ReactMarkdown from 'react-markdown';
import remarkGfm from 'remark-gfm';
import type { SummaryReference } from '@/shared/types/chat';

/**
 * OneChat v5 executive summary with its `[n]` citations, shown above the raw
 * agency sections. Renders nothing when the upstream produced no summary
 * (v4 mode, summary-generation failure, or a smart-fallback answer).
 */
export function SummaryCard({
  summary,
  references = [],
}: {
  summary?: string | null;
  references?: SummaryReference[];
}) {
  if (!summary?.trim()) return null;

  return (
    <div className="mb-3 rounded-xl border border-primary/30 bg-primary/5 px-3 py-2">
      <div className="mb-1 flex items-center gap-1.5 text-xs font-medium text-primary">
        <Sparkles className="h-3.5 w-3.5 shrink-0" />
        สรุป
      </div>
      <div className="prose prose-sm dark:prose-invert max-w-none prose-p:my-1">
        <ReactMarkdown remarkPlugins={[remarkGfm]}>{summary}</ReactMarkdown>
      </div>
      {references.length > 0 && (
        <ul className="mt-2 space-y-0.5 border-t border-primary/20 pt-2 text-[11px] text-muted-foreground">
          {references.map((ref) => (
            <li key={ref.number}>[{ref.number}] {ref.agency_name}</li>
          ))}
        </ul>
      )}
    </div>
  );
}
```
