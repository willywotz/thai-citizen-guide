# Health-based Auto-transition of Agency Status — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Automatically switch an `active` agency to `maintenance` when its 24h error rate exceeds 50%, and switch a rule-set `maintenance` agency back to `active` when it drops below 50%.

**Architecture:** A new `reconcile_statuses()` service reads each agency's 24h error rate from `ConnectionLog`, applies the threshold rule, and persists status transitions. It runs at the end of the scheduler's existing `agency_chat_test` job (every `HEALTH_CHECK_INTERVAL_MINUTES`). A new `Agency.auto_maintenance` boolean distinguishes rule-set maintenance from human-set maintenance so only rule-set agencies are auto-reactivated; the admin status-PATCH endpoint clears that flag on every manual transition.

**Tech Stack:** Python, FastAPI, Tortoise ORM, aerich (migrations), pytest / pytest-asyncio. Tests run on in-memory SQLite via the `db` fixture in `backend/tests/conftest.py`.

**Spec:** `docs/superpowers/specs/2026-06-12-agency-health-auto-status-design.md`

**Working directory for all commands:** `backend/`

**Rule recap:** `error_rate = failures / checks * 100`. With `checks >= 5`: `active` + `error_rate > 50` → `maintenance`; rule-set `maintenance` + `error_rate < 50` → `active`. Exactly 50%, `< 5` checks, or `draft`/`disabled` → no change.

```python
async def error_window(agency_id: UUID) -> tuple[int, int]:
    """Return (checks, failures) over the trailing 24h for an agency."""
    since = now() - timedelta(hours=24)
    rows = await _rows(agency_id, since)
    total = len(rows)
    failures = sum(1 for r in rows if r["status"] != "success")
    return total, failures
```

```python
"""Auto-transition agency status from 24h health.

active + error_rate > 50% (>=5 checks)            -> maintenance (auto-set)
rule-set maintenance + error_rate < 50% (>=5)     -> active
"""
from app.models import Agency
from app.services.agency_health import error_window

_MIN_CHECKS = 5
_THRESHOLD = 50.0


async def reconcile_statuses() -> None:
    agencies = await Agency.filter(status__in=["active", "maintenance"])
    for ag in agencies:
        checks, failures = await error_window(ag.id)
        if checks < _MIN_CHECKS:
            continue
        error_rate = failures / checks * 100
        if ag.status == "active" and error_rate > _THRESHOLD:
            ag.status = "maintenance"
            ag.auto_maintenance = True
            await ag.save(update_fields=["status", "auto_maintenance", "updated_at"])
            print(f"Auto-maintenance: {ag.name} error_rate={error_rate:.0f}%")
        elif ag.status == "maintenance" and ag.auto_maintenance and error_rate < _THRESHOLD:
            ag.status = "active"
            ag.auto_maintenance = False
            await ag.save(update_fields=["status", "auto_maintenance", "updated_at"])
            print(f"Auto-reactivate: {ag.name} error_rate={error_rate:.0f}%")
```
